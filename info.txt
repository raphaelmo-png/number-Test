請幫我建立一個可部署到 Render 的 Python Web App，需求如下。請直接產出完整專案、程式碼、設定檔、部署檔、README，以及必要的測試與操作說明，不要只給我規劃，請直接實作成可執行專案。

【技術目標】
1. 後端使用 Python
2. 使用 FastAPI 作為 Web Framework
3. 使用 Supabase 作為資料庫
4. 前端可先用簡單 HTML + JavaScript，不必使用 React
5. 部署平台使用 Render
6. 專案需可在本機執行，也可直接部署到 Render

【功能需求】
A. 使用者輸入頁
- 建立一個網頁，提供使用者輸入一個數字
- 使用者按下送出後，將數字寫入 Supabase 資料表
- 表單需有基本驗證：
  - 必填
  - 必須為數字
  - 可接受整數
- 送出成功後，顯示成功訊息
- 送出失敗時，顯示錯誤訊息

B. 後台管理頁
- 建立一個 admin 管理頁面
- 可以即時讀取 Supabase 內的資料並顯示最新紀錄
- 至少顯示：
  - id
  - value
  - created_at
- 當數字 >= 55 時，該筆資料要明顯標示「危險」
- 若數字 < 55，顯示正常狀態
- 畫面上要有清楚的狀態樣式區分，例如：
  - 危險：紅色
  - 正常：綠色或一般樣式

C. 即時更新
- 優先使用 Supabase Realtime 讓 admin 頁面即時更新
- 若實作上有困難，可退而求其次使用 polling，但請優先完成 realtime 版本
- 請在程式碼與 README 中清楚說明採用的是 realtime 還是 polling

D. 專案安全性
- 不要把敏感金鑰硬寫在程式碼內
- 所有環境變數統一使用 .env 與 Render environment variables 管理
- 後端與前端的金鑰使用方式要正確，避免暴露 service role key
- 若 admin 頁未做登入，請在 README 說明目前為簡化版，正式上線建議加入登入保護

【資料庫設計】
請在 Supabase 建立一張 table，例如 numbers，欄位至少包含：
- id: bigint / identity primary key
- value: integer not null
- created_at: timestamp with time zone, default now()

3. RLS 是否啟用的建議

【後端 API 需求】
請至少建立以下 API：
1. POST /api/numbers
- 接收數字
- 寫入 Supabase
- 回傳成功或錯誤訊息

2. GET /api/numbers
- 讀取最新資料
- 預設依 created_at 由新到舊排序
- 可限制筆數，例如最新 50 筆

請使用清楚的 API 結構、輸入驗證與錯誤處理。
